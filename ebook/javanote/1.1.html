Java 支持八种基本数据类型：一种布尔类型、一种字符类型、四种整数类型和两种浮点数类型。​​​​

除了基本数据类型之外，Java 还支持引用类型。

1.1.1 布尔类型

布尔类型（boolean）表示真值，只有两个可选值：真或假。Java 使用保留字 true 和 false 表示这两个布尔值。
Java 比其他语言对布尔值的要求严格得多：布尔类型既不是整数类型也不是对象类型，而且不能使用不兼容的值代替布尔类型。
在 Java 中不能使用下面的简写形式：
Object o = new Object();
int i = 1;
if (o) {
  while(i) {
    // ... 
  }
}
Java 强制要求编写简洁的代码，明确表明想做什么比较：
if (o != null) {
  while(i != 0) {
    // ...
  }
}

1.1.2 字符类型

字符类型（char）表示 Unicode 字符。Java 内部使用定长编码 UTF-16BE（16 位）表示字符。
如果想在 Java 程序中使用字符字面量，只需把字符放在单引号中即可。字符字面量可以使用任何一个 Unicode 字符。

字面量是直接出现在 Java 源码中的值，包括整数、浮点数、单引号中的单个字符、双引号中的字符串，以及保留字 true、false 和 null。

Java 还支持一些其他转义序列，用来表示常用的非打印 ASCII 字符。

字符可以转换成整数类型，也可以从整数类型转换而来。字符类型对应的是 16 位整数类型，但它没有符号。

字符类型的包装类 Character 类定义了一些有用的静态方法（static method），用于处理字符，例如 isDigit()、isJavaLetter()、toLowerCase() 和 toUpperCase()。

字符串字面量

除了字符类型之外，Java 还有一种用于处理字符串的 String 类型，不过它不是基本数据类型。因为字符串很常用，所以 Java 提供了一种句法，可以直接在程序中插入字符串。字符串字面量是包含在双引号中的任意文本。例如：
"Hello, World"
"你好，世界\n"
字符串字面量中可以包含能在字符字面量中使用的任何一个转义序列。

1.1.3 整数类型

Java 中的整数类型有 byte、short、int 和 long 四种。所有整数类型都表示有符号的数字，Java 没有 C 和 C++ 中的 unsigned 关键字。
四种类型的字面量形式使用十进制数字，前面还可以加上负号。整数字面量是 32 位 int 类型，如果要表示 64 位 long 类型，就要以 L 或 l 结尾。整数字面量还可以使用十六进制、二进制和八进制形式来表示。
十六进制形式以 0x 或 0X 开头数，使用字母 A 到 F（或 a 到 f）表示数字的十六进制形式。
二进制形式以 0b 开头，当然，只能使用数字 1 或 0。字面量的二进制形式可能很长，所以经常在字面量中使用下划线。

在任何数字字面量中，下划线都会被忽略。下划线纯粹是为了提升字面量的可读性。

八进制形式以 0 开头，而且不能使用数字 8 或 9。这种字面量不常用，应该避免使用。
下面是一些合法的十六进制、二进制和八进制字面量：
0xff           // 使用十六进制表示的十进制数255
0377           // 使用八进制表示的十进制数255
0b0010_1111    // 使用二进制表示的十进制数47
0xCAFEBABE     // 用来识别Java类文件的魔法数
在 Java 中，如果整数运算超出了指定整数类型的范围，不会上溢或下溢，而是直接回绕。例如：
byte b1 = 127, b2 = 1;       // byte类型的最大值是127
byte sum = (byte)(b1 + b2);  // 加法运算的结果直接回绕到-128，即byte类型的最小值
如果发生了这种情况，Java 编译器和解释器都不会发出任何形式的警告。而整数除以零，或者计算除以零后得到的余数，都是非法操作，会抛出 ArithmeticException 异常。

每一种整数类型都有对应的包装类：Byte、Short、Integer 和 Long。这些类定义了一些有用的静态方法，例如 Byte.parseByte() 和 Integer.parseInt()，作用是把字符串转换成整数。

1.1.4 浮点数类型

在 Java 中，实数使用 float 和 double 数据类型表示。浮点数可以以字面量形式插入 Java 程序，其格式为一些可选的数字，后跟一个小数点和一些数字。
浮点数字面量还可以使用指数形式（也叫科学记数法）表示，其格式为一个数后面跟着字母 e 或E 和一个数。第二个数表示 10 的次方，是第一个数的乘数。例如：
1.2345E02    // 1.2345 * 10^2或123.45
1e-6         // 1 * 10^-6或0.000001
6.02e23      // 阿伏加德罗常数：6.02 * 10^23
浮点数默认是 double 类型，若想插入 float 类型的字面量，要在数字后面加上 f 或 F。

浮点数字面量不能使用十六进制、二进制或八进制表示。

float 和 double 类型还能表示四个特殊的值：正无穷大、负无穷大、零和 NaN。浮点数运算的结果超出了所能表示的范围上限，得到的是无穷大；超出了所能表示的范围下限，得到的是零。

NaN，是“Not-a-Number”的简称。浮点数运算不合法得到的就是 NaN。若想检查 float 或 double 值是否为 NaN，必须使用 Float.isNaN() 或 Double.isNaN() 方法。因为 NaN 不等于任何其他数字（包括自己），所以无法用 == 运算符判断。

double inf = 1.0/0.0;          // 无穷大
double neginf = -1.0/0.0;      // 负无穷大
double negzero = -1.0/inf;     // 负零
double NaN = 0.0/0.0;          // NaN
Java 浮点数类型能处理到无穷大的上溢以及到零的下溢，因此浮点数运算从不抛出异常，就算执行非法运算也没事。
无穷大浮点数的表现和设想的一样，例如，无穷大之间的加减运算得到的还是无穷大。负零的表现几乎和正零一样，使用 == 运算符判断负零和正零是相等的。

1.1.5 基本类型之间的转换

Java 允许整数和浮点数之间相互转换。由于每个字符都对应 Unicode 编码中的一个数字，所以字符与整数和浮点数之间也可以相互转换。在 Java 中，布尔值是唯一一种不能和其他基本类型之间相互转换的基本类型。
类型转换有两种基本方式："放大转换"和"缩小转换"。
把小范围类型的值转换成大范围类型的值，称为放大转换。例如，把 int 字面量赋值给 double 类型的变量时，Java 会执行放大转换。
把大范围类型的值转换成小范围类型的值，称为缩小转换。缩小转换并不总是安全的，例如把 13 000 转换成 byte 类型就不合理。缩小转换可能丢失数据，所以试图缩小转换时 Java 编译器会发出警告，就算转换后的值能落在更窄的取值范围内也会警告：
int i = 13;
byte b = i;    // 编译器不允许这么做
不过有个例外，如果整数字面量（int 类型）的值落在 byte 和 short 类型的取值范围内，就能把这个字面量赋值给 byte 或 short 类型的变量。
如果需要强制 Java 执行缩小转换，可以在想转换的值前面加一个括号，在括号里写上希望转换成哪种类型。例如：
int i = 13;
byte b = (byte) i;   // 把int类型强制转换成byte类型
i = (int) 13.456;    // 把double字面量强制转换成int类型，得到的是13

把浮点数转换成整数，浮点数的小数部分会被直接截掉。静态方法 Math.round()、Math.floor() 和 Math.ceil() 执行的是另一些舍入方式。
